# 使用者訪談紀錄

> 本文件記錄 Release 01 使用者訪談的回饋。
> 以下內容由 AI（persona sub-agent）根據 Phase 0 原型模擬產生，尚未經過真實使用者訪談驗證，需人工刪改、補充與判斷。

---

## 訪談對象一：回報者（Reporter）

1. **我想完成的事**
   - 像 M-001「老雜貨店後面需要十幾個人清泥」這樣，快速把聽到的線索送出去，不用先填滿一整張表。
   - 像 M-012 外地家屬一樣，能夠把「我不確定」說清楚，而不是假裝我知道完整狀況。

2. **我看不懂的地方**
   - 在「原始資訊」分頁，每張卡片上都有 `SourceLabel` 標示「社群貼文」「現場回報」，但有些卡片旁邊會跑出黃色「資料品質提示」，我不是很確定那是系統在罵我填錯，還是只是提醒後面的人。
   - 「整理工作台」裡的「建立派遣建議（人事時地物檢查）」按鈕讓我有壓力，我不知道什麼叫「人事時地物」，感覺要填很多東西才能送出。

3. **我最怕的錯誤**
   - 怕點了「送到整理工作台」之後，系統把我只是轉述的話（M-001「有人說」）自動變成「已確認任務」。
   - 怕勾了 `Phase0CandidateForm` 裡的「緊急」之後，資訊還沒補齊就變成紅色的「緊急候選通報」，讓別人以為我已經確認了現場。

4. **我不敢直接相信的資訊**
   - 我自己寫的 M-001 本來就不敢全信，地點只有「老雜貨店後面」，人數也是「十幾個」。
   - 我不敢相信 `detectPhase0Signals` 自動貼在我資料上的黃色標籤一定是對的，例如它可能會把口語誤判成第三方轉述。

5. **我希望系統提醒我的事**
   - 在我送出緊急表單前，跳出提醒：「緊急模式只會讓協作者更快看到，不代表已查核完畢。」
   - 在「原始資訊」卡片上直接告訴我：「你這筆缺少可導航地址／當事人聯絡，要標成不確定再送出嗎？」

6. **我想反問開發者的一個問題**
   - 如果我只是轉述（例如 M-001「有人說」、M-012「外地家屬來電」），系統要怎麼讓後面的人一眼看出我不是親眼確認的人，不會把我當成目擊者？

## 訪談對象二：資訊整理者（Organizer）

1. **我想完成的事**
   - 把 M-005「不知道截圖是哪一天、也不知道是不是官方公告」明確標成需要人工確認，而不是直接變成道路封閉公告。
   - 把 M-009 這種相對完整的現場集合點資訊整理成候選結果，但仍保留「尚未看到官方公告同步更新」的提醒。

2. **我看不懂的地方**
   - 右側「第一階段完成檢查」的五項紅綠燈提示，看起來像是在評分，但我不確定它是在評我的作業完成度，還是在評資料可信度。
   - `Phase0JudgementCard` 裡的「下一步」下拉選項出現 `create_candidate_report`、`send_to_human_review` 等英文字，中文介面裡看到這些值，會讓我不太確定該選哪個。

3. **我最怕的錯誤**
   - 怕「已通過檢查」這個分頁名稱讓行動者以為資料已查核完畢，但文件說它只是通過表單必填檢查。
   - 怕 AI 自動偵測的「資料品質提示」被下一棒當成人工查核過的標記，原文其實沒有被確認。
   - 怕有人把 M-005 的 `sourceType: "official_notice"` 誤認為「官方已確認」，忽略了「沒有官方查證」的提示。

4. **我不敢直接相信的資訊**
   - `messy-reports.json` 裡只有 `id`、`rawText`、`sourceType`、`verificationStatus`、`updatedAt`，沒有當事人聯絡與可導航地址，我不認為這樣就可以直接轉成任務。
   - M-011「志工代長者轉述」我不敢相信，因為還沒確認長者是否同意公開完整地址與接受協助。

5. **我希望系統提醒我的事**
   - 在「第一階段完成檢查」旁邊加一句：「本清單只檢查是否完成整理動作，不代表資料已查核。」
   - 當我選擇「建立派遣建議」時，自動把原始文字的完整內容並排顯示，讓我確認我填進表單的內容不是 AI 幫我補上去的。

6. **我想反問開發者的一個問題**
   - 我整理後的卡片狀態會顯示 `candidate_submitted` 或 `candidate_urgent_submitted`，這些英文詞在行動者眼中會不會太像正式任務狀態？你們有沒有考慮換成更明顯是「未確認候選」的中文標示？

## 訪談對象三：行動者（Actor）

1. **我想完成的事**
   - 在「已通過檢查」分頁快速看到哪些候選通報可以納入行動參考，而不是直接當成執行指令。
   - 判斷我能不能帶雨鞋、水電工或藥品去某一個地點。

2. **我看不懂的地方**
   - 「已通過檢查」頁面的卡片標題寫著「已通過檢查」，但卡片右上角的 `StatusBadge` 卻是「candidate_submitted」或「candidate_urgent_submitted」，我不太確定這到底是可以出發、還是不能出發。
   - `Phase0CandidateForm` 和 `Phase0CandidateGallery` 裡都有「嚴重性：輕／中／重」的顏色排序，但這個嚴重性是誰評的？系統自己算的，還是整理者選的？

3. **我最怕的錯誤**
   - 怕紅色「重（紅）」卡片讓我直覺以為「很危急＝很確定」，結果它只是未確認的緊急候選通報。
   - 怕到現場才發現 M-002「下午還有沒有雨鞋」已經過期，系統沒有提醒我時間有效性。
   - 怕我把「撤銷通報」按鈕當成「取消現場任務」，但 `docs/phase0-observations.md` 說它只是從畫面清單移除。

4. **我不敢直接相信的資訊**
   - M-001 只有「老雜貨店後面」這種無法導航的地點。
   - M-006 有兩個互相矛盾的現場回報：學校側門可以當集合點 vs. 剛剛淹水不適合停留。
   - M-008「A 區先不要再派人」沒說原因，我不敢據此行動。

5. **我希望系統提醒我的事**
   - 在「已通過檢查」頁面最上方放警語：「這裡不是已確認任務清單，只是通過表單檢查的候選資訊。」
   - 對每張卡片明確標示「最後更新時間」與「是否已超過有效期限」。
   - 對有衝突的資訊（如 M-006）顯示：「本筆資料與其他回報不一致，請先人工確認。」

6. **我想反問開發者的一個問題**
   - 當我看到 `candidate_urgent_submitted` 的紅色卡片時，我應該先出發，還是應該先找整理者確認？系統有沒有標示「下一步該找誰」，避免我誤把未確認資訊當成行動指令？
